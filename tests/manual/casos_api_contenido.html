APIs y Solicitudes AJAX

Casos para las funcionalidades que exponen datos al sitio (por ejemplo preguntas y reseñas), explicados con lenguaje amable para quien prueba desde el navegador o herramientas como Postman.

No Nombre/Identificador Descripción Pre condiciones Entradas Pasos Resultados Esperados Pos condición Estado Prioridad
1 CP-API-001 Obtener la lista de preguntas de un producto El endpoint que devuelve preguntas del producto está accesible Parámetro: identificador del producto de prueba 1. Abrir la URL en el navegador o Postman Devuelve un JSON con cada pregunta, autor y fecha La respuesta llega con código 200 Pendiente Media
2 CP-API-002 Enviar una pregunta desde la sección del producto Usuario con sesión iniciada product_id: identificador del producto de prueba, question="¿Tiene envío gratis?" 1. Llamar al endpoint de preguntas desde la interfaz o Postman
2. Revisar la respuesta		 Responde con success=true (éxito) y mensaje "Recibimos tu pregunta" La pregunta aparece como pendiente en la ficha Pendiente Alta
3 CP-API-003 Responder una pregunta como tienda Administrador con acceso (Email="braianoquen@gmail.com", Contraseña="Braian8052@") question_id reciente y answer="Sí, aplica" 1. Llamar al endpoint que guarda la respuesta del vendedor El JSON confirma que se guardó y la respuesta queda marcada como del vendedor El comprador recibe un aviso Pendiente Media
4 CP-API-004 Modificar una pregunta antes de que la respondan El autor es el mismo usuario que edita question_id y texto corregido 1. Llamar al endpoint que permite editar la pregunta Responde con éxito y la pregunta cambia si todavía no había respuesta Queda registro del cambio Pendiente Baja
5 CP-API-005 Quitar una pregunta que todavía no tiene respuesta Pregunta sin respuestas question_id a eliminar 1. Llamar al endpoint que borra la pregunta La respuesta indica éxito y la pregunta desaparece del listado La acción queda documentada Pendiente Baja
6 CP-API-006 Listar las reseñas visibles de un producto Existen reseñas aprobadas product_id: identificador del producto de prueba 1. Llamar al endpoint para listar reseñas de un producto de prueba El JSON incluye calificación, comentario y votos útiles La ficha del producto consume este resultado Pendiente Media
7 CP-API-007 Crear una reseña después de comprar Usuario ya compró el producto product_id: identificador del producto de prueba, rating=5, comment="Excelente" 1. Llamar al endpoint para enviar una reseña Responde con éxito y la reseña queda en revisión Se avisa al administrador para aprobarla Pendiente Alta
8 CP-API-008 Indicar que una reseña ayudó Reseña existente review_id y is_helpful=1 1. Llamar al endpoint que marca una reseña como útil El contador sube y el JSON confirma la acción No permite repetir el voto Pendiente Baja
9 CP-API-009 Buscar productos mientras se escribe en la barra La búsqueda instantánea del sitio está activa Parámetro term="vestido" 1. Consultar el archivo vía navegador o desde la barra real Devuelve productos con nombre, enlace y precio listos para el autocompletado El buscador usa esta respuesta al instante Pendiente Media
10 CP-API-010 Obtener detalles rápidos para el modal del carrito La llamada que devuelve detalles rápidos del producto está disponible product_id: identificador del producto de prueba 1. Abrir la URL con el parámetro indicado Responde con nombre, precios y disponibilidad por color/talla Si no existe muestra un mensaje de error Pendiente Media
11 CP-API-011 Actualizar el estado de pago desde una llamada AJAX La función para actualizar el estado de pago está disponible solo en el panel administrativo order_id: identificador de una orden de ejemplo, status="pagado" 1. Llamar al archivo estando logueado como administrador El JSON confirma que el pago pasó a "paid" (ej. para una orden de ejemplo) La orden registra el cambio en su historial Pendiente Alta
