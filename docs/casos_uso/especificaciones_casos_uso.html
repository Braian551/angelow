Nombre del caso de usoValidar Pago
AutorEquipo Angelow
Fecha2 de Noviembre, 2025
DescripciónVerificar y aprobar pago recibido por transferencia
ActoresAdministrador, Sistema de Pago
PrecondicionesComprobante subido, transacción en estado 'pending'
Flujo Normal1. Admin accede a admin/orders.php y ve órdenes con filtro payment_status='pending'.
2. Hace clic en orden específica, ve detalles en admin/order/detail.php.
3. Descarga comprobante desde ruta almacenada en payment_transactions.payment_proof.
4. Verifica en su banco online que la transferencia fue recibida (monto, referencia, fecha).
5. Si correcto, hace clic en "Aprobar Pago" que ejecuta ajax/admin/update_payment_status.php.
6. Sistema actualiza:
  UPDATE payment_transactions SET status = 'approved', approved_at = NOW(), approved_by = ? WHERE id = ?
  UPDATE orders SET payment_status = 'paid', status = 'processing' WHERE id = ?
7. Genera notificación email al cliente confirmando pago.
8. Libera orden para asignación a delivery.
Flujo AlternativoSi comprobante inválido/fraudulento: marca como 'rejected', notifica cliente, orden cancelada
PosCondicionesPago validado, orden lista para procesamiento, cliente notificado

2. Sistema carga coordenadas destino desde user_addresses.gps_latitude/longitude.
3. JavaScript solicita permiso de geolocalización: navigator.geolocation.getCurrentPosition().
4. Renderiza mapa Leaflet con: marcador origen (ubicación actual), marcador destino, ruta calculada con OSRM.
5. Inicia tracking continuo: navigator.geolocation.watchPosition() envía coordenadas cada 10 seg a delivery/api/update_location.php.
6. Actualiza tabla order_deliveries: location_lat, location_lng, last_location_update.
7. Cambia estado a 'in_transit' si no lo está.
