Total Criterios: 60+
Módulos: 7 módulos principales
HU ID Escenario Título del Criterio Dado que (Contexto) Cuando (Evento) Entonces (Resultado)
HU-01 1 Registro exitoso con email válido Dado que soy un cliente nuevo en la página de registro Cuando completo el formulario con nombre, email válido, teléfono opcional, contraseña (mínimo 6 caracteres) y acepto términos Entonces el sistema verifica que el correo no esté registrado, crea la cuenta, envía un correo de bienvenida y te lleva a tu panel personal
2 Email ya registrado Dado que intento registrarme con un email existente Cuando envío el formulario Entonces el sistema muestra mensaje "Este correo ya está registrado" y mantiene el formulario abierto
3 Datos incompletos Dado que dejo campos obligatorios vacíos Cuando intento enviar el formulario Entonces el sistema muestra "Todos los campos obligatorios deben ser completados"
4 Contraseña muy corta Dado que ingreso una contraseña de menos de 6 caracteres Cuando envío el formulario Entonces el sistema muestra "La contraseña debe tener al menos 6 caracteres"
HU-02 1 Login exitoso Dado que tengo una cuenta registrada Cuando ingreso email y contraseña correctos Entonces el sistema verifica tus datos de acceso, inicia tu sesión y te lleva a tu área personal según tu tipo de usuario
2 Credenciales incorrectas Dado que ingreso datos inválidos Cuando intento iniciar sesión Entonces el sistema muestra "Datos de acceso incorrectos" y registra el intento fallido
HU-03 1 Actualización exitosa Dado que estoy autenticado en mi perfil Cuando modifico datos personales, direcciones o preferencias y guardo Entonces el sistema actualiza la base de datos y muestra mensaje de confirmación
HU-04 1 Solicitud de recuperación Dado que olvidé mi contraseña Cuando ingreso mi email registrado en el formulario de recuperación Entonces el sistema crea un código especial, lo guarda temporalmente y envía un correo con un enlace seguro para cambiar la contraseña
HU-05 1 Búsqueda por texto Dado que estoy en la página de catálogo Cuando ingreso términos de búsqueda Entonces el sistema busca en los nombres, descripciones y categorías de los productos
2 Sin resultados Dado que mi búsqueda no coincide con ningún producto Cuando realizo la búsqueda Entonces el sistema muestra mensaje "No se encontraron productos" y sugiere términos alternativos
HU-06 1 Visualización completa Dado que selecciono un producto del catálogo Cuando accedo a su página de detalles Entonces veo galería de imágenes, descripción completa, precio base, variantes disponibles (tallas, colores), y stock por variante
HU-07 1 Filtros aplicados Dado que tengo resultados de búsqueda Cuando aplico filtros por categoría, género, precio mínimo/máximo, talla o color Entonces el sistema filtra y muestra solo los productos que cumplen con todos los criterios seleccionados
HU-08 1 Agregado exitoso Dado que selecciono producto con variantes específicas Cuando hago clic en "Agregar al carrito" con cantidad válida Entonces el sistema verifica que hay suficiente stock, calcula el precio incluyendo las opciones elegidas, agrega el producto al carrito y muestra un mensaje de confirmación
2 Stock insuficiente Dado que la cantidad solicitada excede el stock Cuando intento agregar al carrito Entonces el sistema muestra "Stock insuficiente" y sugiere cantidad máxima disponible
HU-09 1 Cambio de cantidad Dado que tengo items en el carrito Cuando modifico la cantidad de un producto Entonces el sistema recalcula los precios automáticamente y guarda los cambios
HU-10 1 Descuento válido aplicado Dado que tengo un código de descuento activo y no usado Cuando lo aplico en el checkout Entonces el sistema verifica el código, calcula el descuento (por porcentaje o cantidad fija), actualiza el total y marca el código como utilizado
HU-11 1 Pago completado correctamente Dado que tengo productos en carrito y dirección seleccionada Cuando completo el formulario con número de referencia válido y subo comprobante (JPG/PNG/PDF < 5MB) Entonces el sistema crea un número único de pedido, registra la orden y los productos, guarda el comprobante de pago, marca la transacción como pendiente, vacía el carrito y te lleva a la página de confirmación
2 Comprobante faltante Dado que no subo el archivo de comprobante Cuando intento confirmar el pago Entonces el sistema muestra "Debes subir el comprobante de pago"
3 Archivo inválido Dado que subo archivo con formato no permitido o tamaño excesivo Cuando valido el formulario Entonces el sistema muestra error específico ("El archivo debe ser JPG, PNG o PDF" o "El archivo no puede ser mayor a 5MB")
HU-12 1 Costo calculado automáticamente Dado que tengo dirección de envío seleccionada Cuando el sistema procesa el envío Entonces calcula el costo de envío según las reglas establecidas (según el precio y la zona) y muestra el total actualizado
HU-13 1 Dirección guardada seleccionada Dado que tengo direcciones guardadas en mi perfil Cuando selecciono una para el envío Entonces el sistema la utiliza para cálculo de costos y la guarda en la orden
HU-14 1 Factura generada automáticamente Dado que el pago es aprobado por administrador Cuando se confirma la orden Entonces el sistema crea un comprobante en PDF con los datos fiscales, código QR, detalles de los productos y totales, y lo guarda en el sistema
HU-15 1 Seguimiento en tiempo real Dado que tengo pedidos realizados Cuando accedo a "Mis pedidos" Entonces veo una lista con los estados: pendiente, pagado, preparando, enviando, entregado, con fechas y detalles
HU-16 1 Método registrado correctamente Dado que soy responsable logístico y defino un nuevo método de envío con cobertura, costos y tiempos válidos Cuando guardo la configuración en el panel de envíos Entonces el sistema valida los datos, activa el método para el checkout y lo deja disponible para los cálculos automáticos
HU-17 1 Despacho programado Dado que una orden fue pagada y está lista para salir de bodega Cuando asigno fecha de salida, transportista interno y número de guía Entonces el sistema cambia el estado a "Enviado", registra la trazabilidad y notifica al cliente
HU-18 1 Seguimiento actualizado Dado que administro una orden que se encuentra en tránsito Cuando cambio su estado a "En tránsito" o "Entregado" y adjunto evidencia opcional Entonces el sistema guarda la hora exacta, actualiza el historial y refleja el cambio en el panel del cliente
HU-19 1 Notificación automática Dado que la orden cambió de estado durante el seguimiento Cuando el sistema detecta la transición a "Enviado" o "Entregado" Entonces se envía un correo y una alerta interna al cliente con el nuevo estado y la hora estimada
HU-20 1 Creación de producto Dado que soy administrador autenticado Cuando creo un producto con nombre, descripción, precio, imágenes y variantes Entonces el sistema guarda el producto, sus imágenes y las opciones disponibles, y lo activa en la tienda
2 Edición de producto Dado que tengo un producto existente Cuando modifico sus datos y guardo Entonces el sistema actualiza toda la información relacionada y mantiene la coherencia de los datos
HU-21 1 Actualización de stock Dado que modifico el stock de una variante Cuando guardo los cambios Entonces el sistema actualiza la cantidad disponible y registra el movimiento en el inventario
HU-22 1 Validación de pago Dado que recibo una nueva orden con comprobante Cuando reviso y apruebo el pago manualmente Entonces el sistema marca el pago como aprobado, cambia el estado del pedido a 'preparando' y envía una confirmación al cliente
HU-23 1 Reporte de ventas Dado que solicito reporte por fechas Cuando genero el informe Entonces el sistema revisa las órdenes y productos, calcula los totales y crea un reporte en PDF o Excel con gráficos
HU-24 1 Creación de código promocional Dado que configuro un descuento con código único, tipo (porcentaje/monto), valor, fechas de vigencia y condiciones Cuando lo activo Entonces el sistema guarda el código de descuento y permite usarlo al pagar según las reglas establecidas
HU-25 1 Producto agregado a favoritos Dado que estoy autenticado y visualizo la ficha de un producto con talla y color seleccionados Cuando presiono Guardar en mi lista Entonces el sistema crea el registro en mi wishlist, conserva la variante elegida y actualiza el contador del encabezado
2 Favorito eliminado Dado que tengo productos guardados en mi lista de deseos Cuando hago clic en Quitar o arrastro el elemento a eliminar Entonces el producto desaparece de la lista sin recargar la página y se sincroniza el total mostrado
HU-26 1 Alerta registrada Dado que sigo un producto sin stock o con promoción próxima Cuando activo la opción de avisarme y selecciono correo o notificación interna Entonces el sistema guarda la alerta, muestra una confirmación y la deja pendiente de ejecución
2 Aviso entregado Dado que se actualiza el inventario o inicia la promoción del producto seguido Cuando se cumple la condición configurada Entonces recibo correo y notificación en mi panel con un enlace directo para finalizar la compra
HU-27 1 Listado consultado Dado que tengo avisos pendientes y entro al centro de notificaciones Cuando abro el módulo Entonces se muestran los mensajes ordenados por fecha con iconos de estado y enlaces contextuales
2 Avisos marcados como leídos Dado que quiero limpiar mis pendientes Cuando uso la acción Marcar todo como leído o cierro cada alerta individual Entonces se reinician los contadores y se registra la fecha en el historial del usuario
HU-28 1 Cancelación solicitada Dado que mi pedido sigue en estado Pendiente o Pagado Cuando presiono Cancelar pedido y confirmo la acción Entonces el sistema genera un folio de seguimiento, notifica al equipo interno y me envía el resumen por correo
HU-29 1 Anuncio publicado Dado que defino título, mensaje, fechas y colores para una campaña Cuando guardo el anuncio en el módulo de marketing Entonces aparece en la barra superior con el estilo configurado y se activa solo durante las fechas definidas
2 Slider actualizado Dado que cargo una nueva imagen promocional con enlace y orden específico Cuando confirmo los cambios Entonces la diapositiva queda visible para los clientes y respeta la prioridad establecida
HU-30 1 Nuevo administrador creado Dado que capturo datos personales, rol y permisos de un colaborador Cuando guardo el formulario Entonces el usuario recibe un correo de bienvenida, queda activo y se registra la creación en el historial
2 Administrador desactivado Dado que necesito revocar el acceso de un colaborador Cuando cambio su estado a Inactivo Entonces el ingreso se bloquea de inmediato y el evento queda trazado para auditoría
